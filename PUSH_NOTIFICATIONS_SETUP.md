# Configurazione Notifiche Background (Web Push)

Per far sì che le notifiche arrivino anche quando l'app o il sito sono chiusi, è necessario configurare Web Push tramite Supabase Edge Functions.

## 1. Creazione Tabella Database
Esegui questo SQL nell'Editor SQL di Supabase per creare la tabella dove verranno salvati i token di notifica degli utenti:

```sql
create table if not exists public.push_subscriptions (
  id bigint generated by default as identity primary key,
  user_id uuid references public.utenti(id) on delete cascade, -- Se utenti.id è UUID
  -- Se utenti.id è di tipo diverso (es. bigint), adegua il tipo qui sopra:
  -- user_id bigint references public.utenti(id) on delete cascade,
  subscription_json jsonb not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  unique(user_id)
);

-- Abilita RLS
alter table public.push_subscriptions enable row level security;

-- Policy per permettere agli utenti di inserire/aggiornare la propria sottoscrizione
create policy "Users can manage their own subscriptions"
  on public.push_subscriptions
  for all
  using (true) -- In una vera app con Supabase Auth, usa: auth.uid() = user_id
  with check (true);
```

**Nota di sicurezza:** Poiché questa app usa un sistema di login personalizzato (tabella `utenti` senza Supabase Auth), l'RLS è impostato su `true`. In produzione, assicurati di proteggere questo endpoint o usare Supabase Auth per una sicurezza reale.

## 3. Supabase Edge Function
Crea una nuova Edge Function chiamata `send-push` e incolla il seguente codice (adattandolo se necessario):

```typescript
import { serve } from "https://deno.land/std@0.168.0/http/server.ts"
import { createClient } from "https://esm.sh/@supabase/supabase-js@2"
import WebPush from "https://esm.sh/web-push@3.6.6"

const VAPID_PUBLIC_KEY = "BJJ7e6fpdULY2kA_Fm-pD95CLSE2uqclhPsJPvFtzz55pYjmqOW1Pn1buihWjmntsUPG9YkQbCei0Hu-n21DBx0"
const VAPID_PRIVATE_KEY = Deno.env.get("VAPID_PRIVATE_KEY")!

WebPush.setVapidDetails(
  "mailto:admin@example.com",
  VAPID_PUBLIC_KEY,
  VAPID_PRIVATE_KEY
)

serve(async (req) => {
  const { title, body, url } = await req.json()

  const supabase = createClient(
    Deno.env.get("SUPABASE_URL")!,
    Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
  )

  // Recupera tutte le sottoscrizioni
  const { data: subs, error } = await supabase
    .from("push_subscriptions")
    .select("subscription_json")

  if (error) return new Response(JSON.stringify({ error: error.message }), { status: 500 })

  const notifications = subs.map((s: any) => {
    return WebPush.sendNotification(
      s.subscription_json,
      JSON.stringify({ title, body, url })
    ).catch(err => {
      if (err.statusCode === 404 || err.statusCode === 410) {
        // Rimuovi sottoscrizione scaduta
        return supabase.from("push_subscriptions").delete().eq("subscription_json->>endpoint", s.subscription_json.endpoint)
      }
    })
  })

  await Promise.all(notifications)

  return new Response(JSON.stringify({ success: true }), {
    headers: { "Content-Type": "application/json" }
  })
})
```

### Deploy della funzione:
1. Installa Supabase CLI.
2. Esegui `supabase functions deploy send-push`.
3. Imposta la chiave privata (quella generata al punto 2) come segreto:
   `supabase secrets set VAPID_PRIVATE_KEY=tua_chiave_privata`

## 4. Integrazione nel Pannello Admin
Nell'interfaccia Admin, quando viene inviato un messaggio broadcast, è possibile chiamare questa Edge Function per inviare la notifica a tutti i dispositivi registrati.
